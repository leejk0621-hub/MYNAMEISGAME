# GAME_RULE.md

## 1. 게임 개요 (Game Overview)
*   **게임 이름**: Fruit Catcher (가제)
*   **장르**: 동작 인식 아케이드 게임
*   **목표**: 하늘에서 떨어지는 과일을 바구니로 받아 점수를 획득하고, 폭탄을 피하세요.
*   **플레이 방식**: 웹캠 앞에서 몸을 움직여 바구니를 조종합니다.

## 2. 조작 방법 (Controls)
화면은 3개의 구역(Lane)으로 나뉩니다. 플레이어의 포즈에 따라 바구니가 해당 구역으로 이동합니다.

| 포즈 (Pose) | 동작 설명 | 바구니 위치 |
| :--- | :--- | :--- |
| **LEFT** | 왼쪽으로 기울이기 또는 왼손 들기 | 왼쪽 (Lane 1) |
| **CENTER** | 차려 자세 또는 정면 | 중앙 (Lane 2) |
| **RIGHT** | 오른쪽으로 기울이기 또는 오른손 들기 | 오른쪽 (Lane 3) |

> *참고: Teachable Machine 모델 학습 시 위 3가지 클래스(LEFT, CENTER, RIGHT)를 학습시켜야 합니다.*

## 3. 아이템 및 점수 (Items & Scoring)
위에서 아래로 아이템이 떨어집니다. 바구니가 아이템과 닿으면 판정이 발생합니다.

| 아이템 | 이미지/설명 | 효과 | 점수 |
| :--- | :--- | :--- | :--- |
| **사과 (Apple)** | 🍎 빨간 사과 | 획득 시 점수 추가 | +100점 |
| **바나나 (Banana)** | 🍌 노란 바나나 | 획득 시 점수 추가 | +200점 |
| **황금 사과 (Gold)** | 🌟 반짝이는 사과 | 획득 시 고득점 | +500점 |
| **폭탄 (Bomb)** | 💣 검은 폭탄 | 닿으면 라이프 감소 | **라이프 -1** |

## 4. 게임 규칙 (Rules)
1.  **시작**: `Start` 버튼을 누르면 카운트다운 후 게임이 시작됩니다.
2.  **진행**:
    *   아이템이 랜덤한 위치(Left, Center, Right)에서 생성되어 아래로 떨어집니다.
    *   플레이어는 포즈를 취해 바구니를 이동시켜 과일을 받습니다.
    *   폭탄을 받으면 라이프(하트)가 줄어듭니다.
3.  **레벨 디자인**:
    *   시간이 지날수록 아이템이 떨어지는 속도가 빨라집니다.
    *   또는 일정 점수(예: 1000점)마다 레벨이 오르며 속도가 증가합니다.
4.  **게임 오버 (Game Over)**:
    *   라이프 3개가 모두 소진되면 게임이 종료됩니다.
    *   최종 점수와 도달한 레벨이 표시됩니다.

## 5. UI 구성 (User Interface)
*   **상단 정보창**: 현재 점수(Score), 남은 라이프(Life), 레벨(Level)
*   **게임 화면**: 웹캠 영상 위에 아이템과 바구니(플레이어 캐릭터) 오버레이
*   **게임 오버 화면**: "Game Over" 텍스트, 최종 점수, "다시 시작" 버튼

## 6. 개발 로드맵 (Development Roadmap)
1.  **기본 설정**: 3개 레인(Lane) 구역 설정 및 바구니 이동 구현
2.  **아이템 구현**: 사과, 폭탄 등 아이템 낙하 로직 구현
3.  **충돌 처리**: 바구니와 아이템 간의 충돌 판정 및 점수 계산
4.  **UI 연동**: 점수 및 라이프 표시 업데이트
5.  **게임 루프 완성**: 시작, 종료, 게임 오버 처리
